Introduction Last updated: 2023-03-21

WebXR for WebGL - is a plugin for Unity game engine that helps you to create AR Experiences and run them in web browsers. You can easily create AR experiences or even AR game that interact realistically with objects and environments.

Github Code Example:

You can embed your code snippets using Github gists

Key Features

WebXR for WebGL plugin helps you to create the following AR Experiences:

  • Targets Recognition - for recognition and tracking of prepared images.
  • Targets Capture - for recognition and tracking targets defined by users in runtime.
  • Face recognition - for recognition the position of the face and facial expressions.
  • Surface recognition - for surface recogntion.
  • AR Geolocation - for tracking the direction and distance to geolocations based on the user's location.

Devices Support

Currently, only the Chrome browser on Android devices is fully supported. Some features may not work on iOS devices or other browsers. You can find more details in the features descriptions sections (Targets Recognition, Targets Capture, Face Recognition, Surface Recognition, AR Geolocation ).

Dependencies

Plugin based on MindAR and Three.js

Getting Started

To use the WebXR for WebGL plugin you will need to download the latest release package from Github:https://github.com/Qualium-Systems-Ltd/WebXR-for-WebGL/releases/ and import it to the empty Unity project.

Note

The WebXR for WebGL plugin has been developed with Unity 2021.3.13f1 using the URP (Universal Render Pipeline), but URP is not neccessary. In addition, some features of URP (such as Post-Processing and HDR) is not supported in the plugin.

Installation

  • Open created empty project.
  • Press 'Assets' menu tab and from drop-down list, select option 'Import Package', from next drop-down, select option 'Custom Package...'
  • Select downloaded package and press 'Open'.
  • In the left bottom corner of opened window press 'All' and after that press 'Import' button.
  • Wait for the import process to finish.
  • Done.

Next Steps

Now you can select wich feature you want to implement.

  • Targets Recognition - for recognition and tracking of prepared images.
  • Targets Capture - for recognition and tracking targets defined by users in runtime.
  • Face recognition - for recognition the position of the face and facial expressions.
  • Surface recognition - for surface recogntion.
  • AR Geolocation - for tracking the direction and distance to geolocations based on the user's location.

Or if you have any question check FAQs section.

Targets Recognition

Targets Tracking allows AR apps to detect, track, and augment 2D images. The Multiple Target functionality enables the tracking of more than one image at a time.

Device Support

This function works both on Android (Chrome browser) and iOS devices.

Scene Setup

  • In Hierarchy press 'Righ mouse button (RMB)', from drop-down list select 'AR-Renderer', from next drop-down list select 'Targets Tracking' option.
  • 'TargetsController' game object has been created. As you can see there are some errors in console, but at this stage ignore them.
  • In Hierarchy press 'Righ mouse button (RMB)', from drop-down list select 'AR-Renderer', from next drop-down list select 'Misc', from next drop-down list select 'Target' option.
  • Set created target as child of 'AR-Renderer-Scene' game object. After that select 'TargetController' game object and in Inspector you can see 'Target Track Controller (Script)' attached to it.
  • Drag and drop created target into 'Targets' field of 'Target Track Controller (Script)' component in Inspector. So now 'Targets' field contains two targets.
  • Navigate to 'Assets/Compiler/Targets' folder and put your target image into this folder.
  • Now expand created target in Hierarchy and select 'AR-Renderer-Target-Preview' game object. In Inspector you can see 'Target Preview (Script)' attached to it.
  • Drag and drop your target image into 'Image' field of 'Target Preview (Script)' component. Your target image should appear in Scene view. You can use your target image in Scene view as real-world size reference.
  • For now you can disable default target in Hierarchy.
  • In Hierarchy expand created target and select 'Displayer' game object. Set whatever you want to display as child of this game object. In my case it will be a sphere.
  • Enable default target in Hierarchy if you disabled it earlier.
  • Save your scene. After that open 'File' menu tab and from drop-down list select 'Build Settings...' and add your scene to build.
  • Now when scene is ready we can proceed to Targets Setup section.

Targets Setup

  • In Project overview window navigate to 'Assets/Compiler' and press 'Righ mouse button (RMB)', from drop-down list select 'Show in Explorer' option.
  • In explorer window navigate to '.../Assets/Compiler' folder/ Here you can find 'DotMindCompiler-win' for Windows, 'DotMindCompiler-linux' for Linux and 'DotMindCompiler-macos' for macOS. Run the file depending on your system. In my case it will be 'DotMindCompiler-win'
  • In explorer window navigate to '.../Assets/Compiler' folder/ Here you can find 'DotMindCompiler-win' for Windows, 'DotMindCompiler-linux' for Linux and 'DotMindCompiler-macos' for macOS. Run the file depending on your system. In my case it will be 'DotMindCompiler-win'
  • The compiler page will automatically open in your browser. If this does not happen, go to the address from the console window.
  • As you can see this page is empty. Now we will fix it. Go back to Unity and in Hierarchy select 'TargetController'. In Inspector press 'Generate' button of 'Target Track Controller (Script)' component. If everything is correct the log message will be appear in console. New scheme has been created and saved to 'Assets/Compiler/Schemas' folder
  • Now go back to your browser and press 'Refresh data' button. Now you can see all targets on all of your scenes.
  • On this page you can check if everything is okay and if so, press 'Compile .mind file' button. TThe process will start and you will see the progress under the button.
  • When the comilation is finished, you will see 'Done!' message. Instructions (targets_<<scene index>>.mind) file has been created and saved into your 'Assets/Targets' folder.
  • Compiler's page and console window can be closed.
  • Now when targets are ready we can proceed to Settings section.

Settings

  • Open 'File' menu tab and from drop-down list select 'Build Settings...'
  • Now switch build platform to WebGL
  • Go to 'Player Settings...' and navigate to 'Resolution and Presentation' tab. Select 'AR-Renderer' WebGL Template.
  • If neccessary you can set 'Base href' property. (The base URL to be used throughout the document for relative URLs. Absolute and relative URLs are allowed). For now leave it empty.
  • Navigate to 'Other Settings' tab.
  • Uncheck 'Auto Graphic API'
  • Set 'Lightmap Encoding' to 'Normal Quality'
  • Navigate to 'Publishing Settings' tab.
  • Set 'Compression Format' to 'Disabled'
  • Now we can proceed to Build section.

Build

Everything is ready. It's time to check how it works.

  • Open 'File' menu tab and from drop-down list select 'Build Settings...', in opened window press 'Build' button. Select folder and save your build.
  • To run build locally you will need local webserver with https protocol support. You can download it here.
  • Download and unpack everything from 'local_debug' folder into your build folder.
  • Go to you build folder and run the file depending on your system ('app-windows' for Windows, 'app-linux' for Linux or 'app-macos' for macOS)
  • Open address from the opened console window in browser on your phone.

    • Note

    At this moment only the Chrome browser on Android devices is fully supported.

  • You can use "checker" texture in Scene view as real-world size reference.
  • Save your scene. After that open 'File' menu tab and from drop-down list select 'Build Settings...' and add your scene to build.
  • Now we can proceed to UI Setup section.

  • Drag and drop 'CustomTargetController' game object from Hierarchy into 'Object' field of created option. After that select 'CustomTargetTrackController', and from next drop-down list select 'Capture Target ()'
  • Now we can proceed to Settings section.

Face Recognition

AR Face Recognition lets you render digital 3D content over human faces and recognize their facial expressions to use in the blendshapes system to animate 3D models.

Device Support

This function works both on Android (Chrome browser) and iOS devices.

Scene Setup

  • In Hierarchy press 'Righ mouse button (RMB)', from drop-down list select 'AR-Renderer', from next drop-down list select 'Custom Target Tracking' option.
  • In Hierarchy expand 'FaceController', after that expand 'AR-Renderer-Scene'. Set whatever you what to display as child of this game object. In my case I will use the default facemesh model.
  • You can use facemesh in Scene view as real-world size reference.
  • Now we can proceed to Settings section to test our build, or proceed to Blendshapes section to learn more about the blendshapes system.

Blendshapes

  • In Hierarchy press 'Righ mouse button (RMB)', from drop-down list select 'AR-Renderer', from next drop-down list select 'Misc', from next drop-down list select 'BlendShapesSystem' option.
  • In Hierarchy select 'FaceController'. Drag and drop created 'BlendShapeSystem' into 'AR Blend Shape Controller' field and enable 'Use Blend Shapes' option in Inspector.
  • Done, now you need to create a script that will contain all the settings for this model, you can use the prepared script 'Demo Face Tracking' located in 'Assests/AR-Renderer/Scripts' folder and the settings for it you will find on the demo scene 'FaceTrackingBlendShapesExample'.

    • Note

    Note that in the sample scene all blendshapes that can be processed are set up, if you don't like the performance of a particular blendshape you can try to change the points used there, or the multiplier. But if you are happy with the configuration you can copy all the settings into your script or use the demo script.

  • For more detailed information check 'Blendshapes_Readme' file located in 'Assets/AR-Renderer/Docs' folder.
  • Now we can proceed to Settings section.

Settings

  • Open 'File' menu tab and from drop-down list select 'Build Settings...'
  • Now switch build platform to WebGL
  • Go to 'Player Settings...' and navigate to 'Resolution and Presentation' tab. Select 'AR-Renderer' WebGL Template.

  • Navigate to 'Other Settings' tab.
  • Uncheck 'Auto Graphic API'

Surface Recognition

The surface recognition technology allows you to recognize surfaces and place 3D objects on them. It also allows you to use your device as a controller to move around the Unity scene.

Device Support

Unfortunately, iOS does not have full support for WebXR technology, so it may not work on iOS devices. Stay tuned for future updates.

Scene Setup

  • In Hierarchy press 'Righ mouse button (RMB)', from drop-down list select 'AR-Renderer', from next drop-down list select 'Markers Tracking' option.
  • To test surface recognition we have to do some preparations. Navigate to 'Assets/AR-Renderer/Scripts'. Now select 'MarkerController' game object and drag and drop 'ObjectSpawner' script from 'Assets/AR-Renderer/Scripts' folder into Inspector window.

    • Note

    This controller uses the user's device to move the camera around the Unity scene. If you want to use this option instead of surface recognition, replace the HitPoint game object with an empty game object (Don't forget to set it into 'Hit Point' field of 'MarkerController' game object's, 'MarkerTrackController (Script)' component). To test it place some game objects on the scene. In my case ii will be several primitives. After that you can proceed to Settings section.

  • Create prefab and drag and drop it into 'Prefab' field of the 'Object Spawner (Script)' component. In my case it will be the default cube.
  • Now we can proceed to UI Setup section.

UI Setup

  • Drag and drop 'MarkerController' game object from Hierarchy into 'Object' field of created option. After that select 'ObjectSpawner', and from next drop-down list select 'Spawn Object ()'
  • Now we can proceed to Settings section.

AR Geolocation

AR Geolocation allows developers to create games based on geolocation tracking or attach interactive digital content to geo-based markers.

Device Support

Unfortunately, iOS does not have full support for WebXR technology, so it may not work on iOS devices. Stay tuned for future updates.

Scene Setup

  • In Hierarchy press 'Righ mouse button (RMB)', from drop-down list select 'AR-Renderer', from next drop-down list select 'Geolocation Tracking' option.
  • It's time to add locations. In Hierarchy press 'Righ mouse button (RMB)', from drop-down list select 'AR-Renderer', from next drop-down list select 'Misc', from next drop-down list select 'Location' and Arrow options.
  • In Hierarchy expand 'GeolocationController' and set created 'GeolocationLocation' game object as child of 'Locations' game object.
  • Expand 'ND' game object and set created 'GeolocationArrow' game object as child of 'Arrows' game object.
  • Drag and drop created 'GeolocationLocation' into 'Location' field of created 'GeolocationArrow' game object, 'Arrow (Script)' component.
  • Select created location and set 'name', 'latitude', 'longitude' and if you want to use altitude set altitude and 'Use Altitude' boolean to 'true')
  • Also you can set whatever you want to display as child of created location. In this case game object will appear at your location coordinates.
  • Select 'LocationManager' and drag and drop created location into 'Locations' field and created arrow int 'Arrows' field in Inspector.
  • Now we can proceed to Settings section.
